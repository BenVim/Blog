introduction to Remote Procedure calling

Posted on 2018-05-12 | | Visitors:

Remote procedure calling allows a client to execute procedures on other networked computers or servers.

Client/Server Computing

原书《Power Programming with RPC》

The clinet/server model of computing is a popular model for distributed processing. It applies to any processing environment where one set of entities requests work to be done and another set actually performs the work.

远程与(versus)本地过程调用

对于应用来说，本地与远程调用两者之间有什么不同？实际两者几乎没有什么区别——就是RPC应用程序接口的主要(primary)目标(goal).

图1-1(Figure1-1) 阐述（illustrates）了本地与远程程序（procedure）调用之间的不同，在左侧，调用（客户端）进程在自己的地址空间中执行（executes）一个过程。在右侧，客户端和服务器作为两个独立的进程运行，不再需要在同一台计算机上运行，RPC library 处理（handles）两个进程（processes）之间的通信（communication）。

RPC 使用一种 request-and-reply 通信（communication）模式。客户端程序将请求消息发送到返回应答消息的服务器程序。客户端与服务端进程通过（by means of 译：用或依靠的意思）两个stubs进行通信，一个用户客户端，一个用于服务端。stubs是实现（implement）RPC协议(protocol)的通信(communications)接口（interface），并指定(specify)消息是如何构造(construct)和交换(exchange)的。像ONC RPCGEN这样(typically 译：代表性的，特色的)的协议编译器（compiler）通常用于生成stub。然后stub与客户端程序及服务器程序相连。

stub 包含（contain）将简单的本地过程调用映射(map your..)到一系列(series)网络RPC函数调用的函数。客户端调用其存根中找到的使用RPC库的过程来查找远程进程，然后发出请求。这个远程进程反过来通过（through）服务器stub监听网络。服务器stub执行低级函数，用本地程序调用接口调用（invoke）你的服务例程（routines）。
