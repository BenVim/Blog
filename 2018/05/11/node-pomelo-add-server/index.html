pomelo 添加新的服务模块以及为服务新增组件

Posted on 2018-05-11 | In node | | Visitors:

学习笔记： 先记个大概。

使用pomelo开发的棋牌类游戏，目前的功能完善，并且满足分布式多服务器扩容的需求，在此基础上需要开发牌友圈，同样需要满足分布式的需求，因此，采用的是创建一个牌友圈的服务放在一个主机上，其它的服务器与其通信，确保各服务器数据一致和完整。

遇到的问题：

  1. 新添加的服务未正常启动，报rpcinvoker方法没找到的错误

原因：在server.json里添加了一个公会牌友圈的服务,除了gate，connector，game外新增一个union服力后，没有在adminServer.json中添加该服务的token

adminServer.json

添加后，就正常了。开始出现其它bug，最后查出是rpc调用的传参数不对导致的。

  1. rpc调用的第一个参数是路由规则需要的参数，自定义路由里面可以设计路由规则。

  2. app.rpc.game.gameRemote.onUserConnect(session, uid, sid, cb);

gameRemote 调用的时候，gameRemote是小写。如果写成GameRemote则会不成功。

js里面大多数的是参数错误导致各种bug，仔细点就好了。

  1. rpc调用必须要执行回调函数。否则会报异常错误。
  2. 布署到服务器发现没有正常运行，最后检查是新加的服务的端口没有开导至的。新服务的端口是用域名的，rpc调用，感觉rpc调用端口最好使用ip，减少cdn解析的时间。如果初始化后，是保持连接的，那就没有必要了，用域名更加方便。（是否为初始化后保持连接，还要进下学习rpc原理才可知道）

/////
A服务器的用户加入到B服务器用户创建的房间中，由于内存数据在B服务器，所以A服务器用户注销自己的连接，把自己的areaid改为B服务器用户的，然后再连接服务器，因为后端服务器是使用areaid路由的，所以就可以路由到同一台服务器上去。

这个我有问题，为什么不能在服务端直接完成这个重连操作。而要通过客户端重新发起连接，而且连接的服务器还是A服务器，只是前端服务器connector对后端服务器的路由参数变了。

还有不太明白的就是pomelo的handler连的原理及与rpc的请求原理的差别在哪里。
